1. 클라이언트는 주소의 컴퓨터에 신호를 보낸다. 신호를 받은 서버는 요청한 디렉토리의 코드를 보낸다.
*즉 브라우저는 정보를 요청하고 서버는 정보를 제공한다.

2. 2일차 까지 배운 내용은 브라우저에 관한 내용이다. 3일차는 웹 서버를 다룰 것이다.

3. 서버는 항상 켜져있어야 한다. 웹 서버를 대행해주는 웹 호스팅이 있다. (깃 허브에서도 제공)

4. 2일차 때 활용했던 1, 2, 3, index.html 파일을 활용한다.

5. 해당 repository의 세팅을 master branch로 변경 후 https://web-n.github.io/Web_study/HTML 도메인 주소로 접속.

6. 웹 서버를 위해 Apache라는 소프트웨어를 사용한다.

7. Apache를 통해 웹 서버를 키고 닫을 수 있다.

8. 'localhost'는 '내 컴퓨터'를 의미하는 도메인 이름이다.
*127.0.0.1도 마찬가지

9. http는 HyperText Transfer Protocol의 약자이다. 즉 http는 웹페이지를 전송하기 위해 만든 통신 규약이다.

10. file://는 파일을 직접 열어서 보여준다.

11. 내 컴퓨터 ip를 도메인에 입력하면 다른 컴퓨터에서 내 컴퓨터로 통신할 수 있다.
